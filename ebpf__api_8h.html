[in]sectionOptionally, the name of the section to query.
[in]verboseObtain additional info about the programs.
[out]dataOn success points to a list of eBPF programs.

The list of eBPF programs from this function is TLV formatted as follows:
sections ::= SEQUENCE {
section SEQUENCE of section
}

section ::= SEQUENCE {
name STRING
platform_specific_data INTEGER
count_of_maps INTEGER
byte_code BLOB
statistic SEQUENCE of statistic
}

statistic ::= SEQUENCE {
name STRING
value INTEGER
}

◆ ebpf_program_load()

ebpf_result_t ebpf_program_load ( const char *  file_name,
const ebpf_program_type_t *  program_type,
const ebpf_attach_type_t *  attach_type,
ebpf_execution_type_t  execution_type,
struct bpf_object **  object,
fd_t *  program_fd,
const char **  log_buffer 
)

Load eBPF programs from an ELF file based on default load attributes. This API does the following:

  1. Read the ELF file.
  2. Create maps.
  3. Load all programs.
  4. Return fd to the first program.

If the caller supplies a program type and/or attach type, that supplied value takes precedence over the derived program/attach type.

Parameters
[in]file_nameELF file name with full path.
[in]program_typeOptionally, the program type to use when loading the eBPF program. If program type is not supplied, it is derived from the section prefix in the ELF file.
[in]attach_typeOptionally, the attach type to use for the loaded eBPF program. If attach type is not supplied, it is derived from the section prefix in the ELF file.
[in]execution_typeThe execution type to use for this program. If EBPF_EXECUTION_ANY is specified, execution type will be decided by a system-wide policy.
[out]objectReturns pointer to ebpf_object object. The caller is expected to call bpf_object__close() at the end.
[out]program_fdReturns a file descriptor for the first program. The caller should not call _close() on the fd, but should instead use bpf_object__close() to close this (and other) file descriptors.
[out]log_bufferReturns a pointer to a null-terminated log buffer. The caller is responsible for freeing the returned log_buffer pointer by calling ebpf_free_string().
Return values
EBPF_SUCCESSThe programs are loaded and maps are created successfully.
EBPF_INVALID_ARGUMENTOne or more parameters are incorrect.
EBPF_NO_MEMORYOut of memory.
EBPF_ELF_PARSING_FAILEDFailure in parsing ELF file.
EBPF_FAILEDSome other error occured.
